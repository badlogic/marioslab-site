When I decided to throw everything away and start from scratch, I was looking through available options for building a (mostly) static site. My requirements for the new site:

  • All of the content must be editable via any text editor.
  • I must be able to freely choose the structure of the content.
  • Must provide an expressive templating engine so I can stay in text files for the most part.
  • Allow me to extend the functionality of the templating engine with code in a typed language.
  • Don't force some theming format on me. I want to control all of the CSS and keep it minimal.
  • Let me stuff everything in a Git repository.
  • Bonus points if I can add a HTTP RPC web service for dynamic content such as comments, preferably in the same typed language with which I can extend the template engine.

As a first step, I checked out the solutions my peers went with. Andre is using Jekyll. Michael used to use Jekyll for his site, but recently switch to Hugo. I'll refrain from citing specific reasons. Suffice it to say that a lot of swearing and phrases like "bit rot" were involved.

I went to check out both Jekyll and Hugo. Jekyll is built on Ruby, uses Liquid as its (capable) templating engine, and dictates a very rigid directory structure. I'm not a big fan of Ruby (fight me), and while Jekyll let's you do a lot, I felt I'd probably be fighting it's rigidity. Jekyll's documentation was fantastic though!

Hugo comes off as a more enterprise-y version of Jekyll. That also expresses itself in its documentation. There's a lot of it, including videos. Why people think videos are a great source of documentation I'll never understand. Hugo is built on Go and uses Go's not very expressive, built-in templating engine. Just based on the overwhelming documentation, Hugo seemed to complex to me, so I abandoned it as well.

How hard can it be?
